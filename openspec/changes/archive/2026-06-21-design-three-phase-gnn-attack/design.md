## 上下文

当前 `gnn_attack/` 模块实现了 GNN 边预测的预训练+推理管道，但有两个关键缺失：（1）让模型适应目标加密数据分布的自训练机制；（2）从推断的边重建几何形状的布局算法。本设计定义完整的三阶段方案，产出概要设计和详细设计两份文档。

## 目标 / 非目标

**目标：**
- 定义三阶段的完整数据流：每个阶段拿到什么数据、数据从哪来、输出什么
- 定义三阶段之间的接口契约
- 给出关键算法的原理和伪代码

**非目标：**
- 不定义具体 Python 实现
- 不修改现有代码

### 交付物结构

```
docs/
├── design-three-phase-summary.md    ← 概要设计（全景图 + 对比表）
└── design-three-phase-detailed.md   ← 详细设计（算法 + 接口 + 验证）

后续 gnn-attack 代码实现强制关联:
  设计文档中定义的接口函数、数据格式、验证指标
  → 代码实现必须严格对齐
```

### 攻击者能力边界

```

│                        攻击者数据全景                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌─── 可获取 ───────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  加密数据:                                                    │   │
│  │    每条记录 = (token, Enc(坐标))                              │   │
│  │    → 可区分不同记录，但不知道坐标                              │   │

│  │    每次范围查询 → 返回满足条件的 token 列表                    │   │
│  │    → 可计算: 共现矩阵 C[i,j] = token_i 和 token_j 共现次数    │   │
│  │                                                              │   │
│  │  共现矩阵 C[N,N] 是攻击者唯一的可用信号                        │   │

│                                                                     │

│  │  任何一对记录是否真实相邻 (标签)                                │   │
│  │  加密密钥（无法解密）                                          │   │
│  │                                                              │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                                                                     │

│  │  → 模拟加密查询 → 计算共现矩阵                                │   │
│  │  → 有坐标所以有真实标签（哪些对相邻）                          │   │
│  │  → 用作训练数据                                                │   │
│  │                                                              │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                                                                     │

│  │  加密数据在 2D 平面上的几何分布形状                            │   │
│  │  = 每个 token 的推测 (x, y) 坐标                               │   │
│  │                                                              │   │
│  └──────────────────────────────────────────────────────────────┘   │
```

### 三阶段数据流：各阶段拿到什么、产出什么

```
┌──────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  Phase 1 预训练                                                      │
│  ─────────────                                                       │
│  数据来源: 攻击者自己生成（虚构点集 + 模拟查询）                       │
│  输入:   无（生成器内部自给自足）                                     │
│  标签:   有（因为坐标是攻击者自己生成的）                              │
│  输出:   预训练 GNN 模型参数                                         │
│                                                                      │
│           ┌──────────────────┐                                       │
│   合成数据 │ 形状×N×采样率    │                                       │
│   ──────→  │ 栅格/随机/轮廓  │──────→  GNN_pre (模型参数)             │
│           │ N=20~800        │                                       │
│           │ p=1%~50%        │                                       │
│           └──────────────────┘                                       │
│                                                                      │
│  ───────────────────────────────────────────────────────────────     │
│                                                                      │
│  Phase 2 自训练                                                      │
│  ─────────────                                                       │
│  数据来源: 目标加密服务的实际查询响应（攻击者监听到的）                 │
│  输入:   目标共现矩阵 C_target[N,N]（从加密查询响应计算）              │
│         + Phase 1 输出的 GNN_pre                                      │
│  标签:   无真实标签 — 使用模型自己的高置信预测作为伪标签               │
│  输出:   微调后的 GNN_fine 模型参数                                   │
│         + 推断的边集合 E_hat = {(i,j,prob)}                           │
│                                                                      │
│   目标加密查询响应                                                    │
│   ──────────────────→ 共现矩阵 C_target[N,N]                          │
│                              │                                       │
│                              ▼                                       │
│           ┌──────────────────────────────────────┐                   │
│           │  迭代自训练循环:                      │                   │
│           │  1. GNN(C_target) → 边概率            │                   │
│           │  2. Top-K 高置信边 → 伪正标签          │                   │
│           │  3. 用伪标签微调 GNN                  │                   │
│           │  4. 收敛? → 输出 E_hat               │                   │
│           │    否则 → 回到步骤 1                  │                   │
│           └──────────────────────────────────────┘                   │
│                    │                                                 │
│                    ▼                                                 │
│           GNN_fine + E_hat                                           │
│                                                                      │
│  ───────────────────────────────────────────────────────────────     │
│                                                                      │
│  Phase 3 形状重建                                                    │
│  ─────────────                                                       │
│  数据来源: Phase 2 输出的边集合                                       │
│  输入:   边集合 E_hat = {(i,j,prob)}                                  │
│         （可选）少数已知锚点坐标                                       │
│  标签:   不需要                                                       │
│  输出:   每个 token 的推测 2D 坐标 pos[N,2]                           │
│         = 攻击者想要的「加密数据形状」                                 │
│                                                                      │
│   E_hat                                                              │
│   ──────→ 力导向布局（弹簧-斥力模型）                                 │
│                    │                                                 │
│                    ▼                                                 │
│           pos[N,2] ← 推测的 2D 点分布                                 │
│                    │                                                 │
│                    ▼                                                 │
│           可视化 / 评估                                                │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

## 决策

### D1. 两文档交付模型

**概要设计文档 (`design-three-phase-summary.md`)** 回答以下问题：
- 这个攻击是什么？（场景、目标、输入输出全景）
- 三阶段分别做了什么？（数据流图、关键步骤）
- 与原始方法和当前方案有什么区别？（对比表）
- 能否工作？（假设前提、风险、验证标准）

**详细设计文档 (`design-three-phase-detailed.md`)** 回答以下问题：
- 每个阶段的完整算法流程（伪代码）
- 各阶段的接口契约（输入格式、输出格式、约束条件）
- 验证标准和边界条件
- Phase 1→Phase 2、Phase 2→Phase 3 的数据格式约定

```mermaid
代码实现 ←──强制关联──→ 设计文档
  │                        │
  │  函数签名必须对齐       │  接口契约定义
  │  数据格式必须对齐       │  数据格式定义
  │  验证指标必须对齐       │  验证标准定义
  │                        │
  └──────── 双向追踪 ──────┘
```

**理由：** 避免历史问题重演——之前的 GNN 实现虽然有计划文档，但代码实际行为与文档描述脱节（训练 vs 推理 k-NN 图构建不一致、特征含义变化未更新文档等）。两份文档分别面向「理解全景」和「指导实现」，所有接口在详细设计中显式定义。

### D2-D5：与已有 design.md 一致（三阶段架构、预训练多样性、自训练机制、力导向布局、共现图消息传递）

## 风险 / 权衡

| 风险 | 缓解 |
|------|------|
| Phase 2 伪标签质量差导致发散 | Phase 1 训练包含 N≥200 场景确保 AUC>0.7；自训练 lr≤预训练 lr/10 |
| 力导向布局丢失方位信息 | 不求绝对方位，求拓扑同胚。有锚点时 Procrustes 对齐 |
| 设计文档与代码脱节 | 详细设计定义显式接口契约，代码实现 review 时逐项对照 |
