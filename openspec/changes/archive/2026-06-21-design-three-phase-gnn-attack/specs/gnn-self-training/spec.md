## 新增需求

### 需求: 伪标签筛选策略

Phase 2 自训练必须从模型自身的高置信预测中选取伪标签，禁止使用任何地面真值。

#### 场景: 正样本伪标签

- **当** Phase 1 预训练 GNN 在目标共现矩阵上完成首次推理
- **那么** 选取边概率最高的 Top-K 对（K ≈ 2×N）作为伪正样本
- **而且** 这 K 对边必须覆盖图中不同区域（避免集中在某个局部子图）

#### 场景: 负样本伪标签

- **当** Phase 1 预训练 GNN 在目标共现矩阵上完成首次推理
- **那么** 选取边概率最低的 Bottom-L 对（L ≈ 10×N）作为伪负样本
- **而且** 排除伪正样本和与其共现高度重叠的边（避免将潜在邻居误标为负）

#### 场景: 不确定区域放弃

- **当** 边概率处于中间区域
- **那么** 该边不参与自训练的损失计算
- **而且** 不确定区域的大小为总边数的 60%-80%

### 需求: 迭代微调

自训练必须是迭代的，每轮用更新后的模型重新推理并更新伪标签。

#### 场景: 微调步骤

- **当** 每一轮自训练迭代
- **那么** 用当前伪标签对预训练 GNN 做小幅度参数更新（学习率 ≤ 预训练学习率的 1/10）
- **而且** 只更新边预测器和最后层消息传递参数（冻结编码器）

#### 场景: 一致性正则化

- **当** 计算自训练损失
- **那么** 对目标共现矩阵做随机扰动（屏蔽 5% 的响应重新计算 C'）
- **而且** 要求 GNN 对 C 和 C' 输出一致的边概率（L2 距离作为正则项）

#### 场景: 收敛判定

- **当** 连续两轮自训练的 Top-K 伪正样本边集合重合率 ≥ 90%
- **那么** 自训练终止
- **而且** 取最后一轮选取的 Top-2N 概率边作为 Phase 2 的最终输出边集合

### 需求: 输入输出约束

Phase 2 只依赖 Phase 1 的模型和目标的共现矩阵，不依赖源代码或数据集特定逻辑。

#### 场景: Phase 2 输入

- **当** Phase 2 启动
- **那么** 输入必须仅为：预训练 GNN 的模型参数 + 目标加密数据的共现矩阵 C[N,N]
- **而且** 禁止使用任何明文坐标、真实标签或数据集特定的加载逻辑

#### 场景: Phase 2 输出

- **当** Phase 2 完成
- **那么** 输出必须为：微调后的 GNN 参数 + 推断的边集合 E_hat = {(i,j, prob)}
- **而且** E_hat 中的 prob 来自最后一轮模型的预测概率
