## 新增需求

### 需求: 多样化合成训练数据

Phase 1 预训练数据必须覆盖多种形状、多种点数和多种采样率的组合，以确保模型学习到与目标分布无关的通用规律。

#### 场景: 形状覆盖

- **当** 生成预训练数据
- **那么** 训练样本必须包含以下形状类型：规则栅格（grid）、随机点云（random）、圆形轮廓（circle）、L形轮廓（l-shape）、不规则轮廓（contour）
- **而且** 每种形状至少有 10% 的样本占比

#### 场景: 点数覆盖

- **当** 生成预训练数据
- **那么** 训练样本必须覆盖 N 从 20 到 800 的范围
- **而且** 至少包含小 N（20-50）、中 N（60-150）、大 N（200-800）三个尺度各有至少 20% 的样本

#### 场景: 采样率覆盖

- **当** 模拟加密查询的采样过程
- **那么** 训练样本的响应采样率必须覆盖 1% 到 50% 的范围
- **而且** 每个样本的采样率从该范围中随机选取

### 需求: 预训练验证标准

Phase 1 完成后必须通过验证评估才能进入 Phase 2。

#### 场景: 验证集 AUC

- **当** 预训练完成
- **那么** 在独立的验证集（含 N≥200 的大场景）上计算边预测的 AUC 指标
- **而且** AUC 必须 > 0.7 才视为预训练成功，可以进入 Phase 2

#### 场景: 多阈值 Precision/Recall

- **当** 预训练完成
- **那么** 在验证集上输出多个阈值下的 Precision 和 Recall
- **而且** 该结果作为 Phase 2 自适应阈值选择的参考基线

### 需求: 共现图消息传递

GNN 的消息传递图必须直接基于共现矩阵构建，而非基于共现向量的 k-NN 筛选。

#### 场景: 共现加权图

- **当** GNN 执行消息传递
- **那么** 消息传递图必须包含所有共现 > 0 的点对作为边
- **而且** 每条消息传递边的权重为归一化后的共现值（高共现 → 高消息权重）

#### 场景: 注意力聚合

- **当** 在共现图上进行邻居消息聚合
- **那么** 聚合函数必须使用基于共现值的注意力权重
- **而且** 高共现邻居的消息贡献大于低共现邻居
