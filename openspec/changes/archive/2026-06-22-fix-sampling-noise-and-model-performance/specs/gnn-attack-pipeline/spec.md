# gnn-attack-pipeline（增量修改）

## 修改需求

### 需求：采样率通过随机噪声实现

ID: `req-sampling-noise`

采样率 `--p` 参数必须通过向共现矩阵加二项随机噪声来实现，而非简单的乘因子后再归一化。

#### 场景：p=100 时无噪声

- **当** 用户指定 `--p 100`
- **那么** C_target 不加任何噪声
- **且** 结果与当前 p=100 完全相同

#### 场景：p<100 时引入二项噪声

- **当** 用户指定 `--p N`（N<100）
- **那么** C_target 的每个元素 c_ij 加相对噪声 σ_rel = sqrt((100-p)/(p * c_ij + ε))
- **且** 噪声服从正态分布 N(0, σ_rel × c_ij)
- **且** 加噪后 clip 到 [0, ∞)，再逐行归一化

#### 场景：低 p 产生更大方差

- **当** p=10
- **那么** 归一化前 C_target 的相邻点对之间的共现值波动 > p=50 的波动
- **且** 归因于二项分布的方差公式 Var = c_ij × p/100 × (1 - p/100)
