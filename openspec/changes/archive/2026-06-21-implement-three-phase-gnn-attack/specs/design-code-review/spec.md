## 新增需求

### 需求: 设计-代码追溯表

`docs/design-code-review.md` 必须记录代码实现与设计文档详细设计的逐接口比对结果。

#### 场景: 接口追溯

- **当** 每个实现模块完成后
- **那么** 在追溯表中记录：接口名称、设计文档章节、代码位置、对齐状态（✓/✗/⚠）、备注

#### 场景: 偏差记录

- **当** 代码实现与设计文档存在偏差
- **那么** 在追溯表中标记 ✗ 或 ⚠
- **而且** 说明偏差原因和是否需要更新设计文档

#### 场景: 完整性检查

- **当** 所有实现模块完成后
- **那么** 遍历追溯表确认设计文档详细设计末尾「代码实现关联要求」中列出的 11 个接口点全部有对应记录
- **而且** 所有记录状态为 ✓ 或 ⚠（有合理说明）
