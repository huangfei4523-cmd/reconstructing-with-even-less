## 修改需求

### 需求: 预训练数据生成与模型训练

Phase 1 必须在参数空间覆盖的合成数据上训练 GNN，使其学会「共现模式→空间相邻」的通用映射。

#### 场景: 参数空间网格

- **当** 生成预训练数据
- **那么** 必须覆盖三维参数空间：N(点数)=20~800、形状=grid/random/circle/contour、p(采样率)=1%~50%
- **而且** 每种配置生成 5-10 个样本，总计约 1500 个训练样本

#### 场景: 消息传递图使用真正的共现矩阵

- **当** 训练或验证循环的每次 batch 处理
- **那么** 消息传递图 `edge_index` 必须通过 `build_cooc_message_graph(cooc)` 构建，其中 `cooc` 是从 `CooccurrenceDataset` 获取的真正的共现矩阵
- **而且** 边特征 `edge_feat` 必须通过 `extract_edge_features(cooc, responses, edge_index)` 计算
- **而且** 禁止使用 `node_feat[:,:3] @ node_feat[:,:3].T` 作为 `cooc` 的替代

#### 场景: 训练和验证使用一致的边特征

- **当** 验证循环执行前向传播
- **那么** 传入的 `edge_features` 参数不得为 `None`，必须与训练循环一致地传入通过 `extract_edge_features(cooc, responses, edge_idx)` 计算的值
- **而且** 验证循环的消息传递图构建方式与训练循环完全相同

#### 场景: 验证标准

- **当** 预训练完成
- **那么** 验证集 entropy < 0.60（当前 0.693 = 随机）
- **而且** 验证集最佳 F1 > 0.30
- **而且** 大 N 场景（≥200）的验证指标不低于小 N 场景

#### 场景: 反馈闭环

- **当** 某个 N/p 配置的验证 F1 < 0.30
- **那么** 对该配置增补样本并重新训练
- **而且** Phase 2 自训练发散时回到 Phase 1 补充训练数据多样性
