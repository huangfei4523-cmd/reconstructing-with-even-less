### 需求: 攻击者能力边界定义

攻击者（不可信服务器提供者）必须明确其可获取和不可获取的信息。

#### 场景: 可获取的信息

- **当** 攻击者作为服务器提供者运行可搜索加密服务
- **那么** 攻击者可以获取存储的所有加密数据记录（token + 密文）
- **而且** 攻击者可以监听到每次范围查询请求和对应的加密响应（包含满足查询条件的记录 token 列表）
- **而且** 攻击者可以从所有监听到的响应中计算任意两记录的共现次数 → 共现矩阵 C[N,N]

#### 场景: 不可获取的信息

- **当** 攻击者尝试重建数据分布
- **那么** 攻击者无法获取任何记录的明文坐标
- **而且** 攻击者无法获取任何一对记录的真实邻接关系（标签）
- **而且** 攻击者无法解密响应中的 token（只能观测 token 的共现模式）

#### 场景: 攻击者可生成的信息

- **当** 攻击者进行预训练
- **那么** 攻击者可以自由生成任意形状、任意规模的点集并模拟加密查询过程
- **而且** 攻击者可以生成对应的地面真值邻接标签（因为坐标是自己生成的）
