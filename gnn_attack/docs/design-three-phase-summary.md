# 三阶段可搜索加密 GNN 攻击 — 概要设计

## 1. 攻击场景全景

### 攻击者身份

不可信的服务器提供者：存储加密后的所有数据，可以监听每一次客户端查询请求及加密响应。

### 攻击者能拿到什么

```
┌─────────────────────────────────────────────────────────────────────┐
│                        攻击者数据全景                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ✓ 可获取                                                        │
│  ├─ 加密数据记录: {token_001, Enc(x₁,y₁)}, {token_002, Enc(x₂,y₂)}…│
│  ├─ 每次范围查询响应: 满足条件的 token 列表                         │
│  └─ 由此可计算: 共现矩阵 C[N,N]，C[i,j] = token_i 和 token_j       │
│                  在多少条查询响应中同时出现                           │
│                                                                     │
│  ✗ 不可获取                                                          │
│  ├─ 任何记录的明文坐标 (x,y)                                       │
│  ├─ 任何一对记录的真实邻接关系（标签）                                 │
│  └─ 加密密钥和加解密能力                                             │
│                                                                     │
│  ★ 可任意生成（攻击者自己造的训练数据）                                 │
│  ├─ 任意形状的点集 + 已知坐标                                       │
│  ├─ 模拟加密范围查询                                                │
│  └─ 因为有坐标 → 知道哪些对是相邻的（有训练标签）                      │
│                                                                     │
│  → 最终想要的输出:                                                    │
│  每个 token 在 2D 平面上的推测位置 → 重建的加密数据空间分布形状         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. 三阶段完整数据流

```
┌──────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  Phase 1 预训练                                                       │
│  ─────────────                                                        │
│                                                                       │
│  Step 1 — 定义参数空间：                                               │
│    攻击者不需要知道目标形状，但必须覆盖影响「共现→相邻」的三个维度:        │
│    维度 1: 点数 N（20~800） → 共现稀疏度变化                             │
│    维度 2: 形状拓扑（栅格/随机/圆形/轮廓）→ 正负比变化                    │
│    维度 3: 采样率 p（1%~50%）→ 信噪比变化                               │
│    210 种配置 × 5-10 样本 ≈ 1500 个训练样本                              │
│                                                                       │
│  Step 2 — 生成合成数据：                                               │
│    for 每种配置 in 参数空间:                                            │
│      生成点集坐标 → 模拟加密查询 → 采样 → 共现矩阵 → 标签(曼哈顿≤1)       │
│                                                                       │
│  Step 3 — 训练 GNN:                                                    │
│    用 Step 2 生成的数据训练 GNN，学「共现模式→空间相邻」                   │
│    验证: 留出集 AUC > 0.7                                              │
│                                                                       │
│  反馈: 如果某个 N/p 配置 AUC 低 → 回到 Step 2 对该配置增补样本            │
│  如果 Phase 2 自训练发散 → 回到 Step 1 补充训练数据多样性                 │
│                                                                       │
│  数据来源: 攻击者自己生成（虚构点集 + 模拟查询）                          │
│  训练标签: ✓ 有（坐标是攻击者自己造的，已知相邻关系）                       │
│  输入:   无（生成器内部提供）                                            │
│  输出:   预训练 GNN 模型参数 (checkpoint)                                │
│                                                                       │
├──────────────────────────────────────────────────────────────────────►│
│                                                                       │
│  Phase 2 自训练                                                        │
│  ─────────────                                                        │
│  数据来源: 目标加密服务的查询响应 → 计算共现矩阵 C_target[N,N]           │
│  训练标签: ✗ 无 — 用 GNN_pre 的高置信度预测作为伪标签                   │
│  输入:   C_target[N,N] + GNN_pre                                      │
│  输出:   GNN_fine 模型参数 + 推断边集合 E_hat = {(i,j,prob)}            │
│                                                                       │
│   C_target ──→ [GNN推理→选伪标签→微调] ──→ 收敛? ──→ GNN_fine + E_hat │
│                 └──← 否 ← 重新推理 ──←──┘                              │
│                                                                       │
├──────────────────────────────────────────────────────────────────────►│
│                                                                       │
│  Phase 3 形状重建                                                       │
│  ─────────────                                                        │
│  数据来源: Phase 2 输出的 E_hat                                       │
│  训练标签: 不需要                                                      │
│  输入:   E_hat = {(i,j,prob)}                                        │
│  输出:   pos[N,2] 推测的 2D 点坐标 → 重建形状                          │
│                                                                       │
│   E_hat ──→ 力导向布局 (弹簧-斥力) ──→ pos[N,2] ──→ 可视化             │
│                                                                       │
└──────────────────────────────────────────────────────────────────────┘

各阶段输入输出摘要:

  Phase │ 数据来源      │ 输入            │ 有标签? │ 输出
  ──────┼───────────────┼─────────────────┼─────────┼──────────────────
   1    │ 攻击者自生成    │ —               │    ✓    │ GNN_pre
   2    │ 目标加密查询    │ C[N,N]+GNN_pre   │    ✗    │ GNN_fine + E_hat
   3    │ Phase 2 输出    │ E_hat           │  不需要  │ pos[N,2]
```

## 3. 与已有方案的对比

| 维度 | 原始方法（论文） | 当前 GNN 方案 | 三阶段 GNN 方案 |
|------|-----------------|--------------|----------------|
| **信息利用** | 集合交集（仅 size=2） | 全共现矩阵的统计特征 | 全共现矩阵 + 图消息传递 |
| **数据依赖** | 仅目标共现 | 合成(小N)训练 + 目标推理 | 合成(大范围N)预训练 + 目标自训练 |
| **训练阶段数** | 0 | 1 | 2（预训练+自训练） |
| **输出类型** | 边集合（拓扑） | 边集合（拓扑） | **2D 坐标（几何）** |
| **假设前提** | 存在 size=2 的响应 | 预训练分布能泛化 | 预训练 AUC>0.7 可启动自训练 |
| **已知限制** | p低时 size-2 极少 | cali_50 上 0 条边 | 自训练可能发散需正则化 |
| **需要目标标签** | 否 | 否 | 否（Phase 2 用伪标签） |
